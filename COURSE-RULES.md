# Course Rules

---

# Attendance

Attendance is expected for every scheduled session.

If a trainee misses a session,

they should complete:

- Slides
- Recording
- Lab
- Homework

before attending the next session.

---

# Hands-on Labs

Every session includes a mandatory lab.

A lab is considered complete only after meeting the Definition of Done.

---

# Homework

Homework should be completed before the following session.

Late submissions may receive feedback only.

---

# Git Workflow

Every trainee works in their own repository.

Workflow:

Fork

↓

Clone

↓

Implement

↓

Commit

↓

Push

↓

Pull Request

↓

GitHub Actions Validation

---

# Commit Messages

Use descriptive commit messages.

Example

```
session-01: add-product-service-eureka-config
```

---

# Definition of Done

A task is complete when:

- Code builds successfully
- Tests pass
- Acceptance Criteria satisfied
- No compilation errors
- Commit pushed
- Pull Request created

---

# Code Quality

Follow:

- Clean Code
- Constructor Injection
- REST naming conventions
- Meaningful commit messages

---

# Academic Integrity

Learning is collaborative.

Code submission is individual.

Copying solutions defeats the purpose of the lab.

---

# Questions

Questions are encouraged during every session.

Discussion is part of the learning process.
